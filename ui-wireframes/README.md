## ① Demo 简介

**一句话形态**：本地优先的桌面效率助手。

**核心用户**：知识工作者 / 程序员。

**它做什么**：Shadow Worker 常驻系统托盘，在你工作时默默完成三件事——

1. **自动记账**：按白名单追踪前台应用（Cursor / Chrome / 微信…），自动归类为 编程 / 浏览 / 沟通 / 办公，生成时间轴、热力图和类别占比，回答「我今天到底做了什么」。
2. **语音输入**：全局热键长按即录，ASR 实时转写，LLM 自动去口语化、补标点，结果可预览或一键注入当前焦点输入框。
3. **AI 摘要**：VLM 定时/按热键截图理解当前屏幕，自动写入工作记录；并通过 MCP 服务把工作日志暴露给 Claude / Cursor 等 Agent，让 AI 直接帮你生成日报周报。

**核心功能一览**

| 模块 | 能力 |
|------|------|
| 🎙️ 语音输入 | 长按/点按热键录音 · ASR 实时转写 · LLM 自动润色 · 预览/直接注入 |
| 📊 行为采集 | 应用白名单 · 自动归类 · 时间轴 · 热力图 · 类别占比 · 应用排行 |
| 👁️ VLM 屏幕理解 | 定时/按需截图 · 视觉模型理解屏幕 · 自动写入工作记录 |
| ⚡ 快捷提示词 | Ctrl+数字 一键注入预设提示词（翻译 / 会议纪要 / 润色） |
| 🔌 MCP 工具服务 | 对 Agent 暴露 get_worklog / get_summary / search_events 等工具 |
| 🔒 本地优先 | SQLite 本地存储 · ASR/VLM/LLM 云端或本地模型自由切换 |

**技术栈**：Go 后端 · Qt6/QML 前端 · gRPC + Protobuf · whisper.cpp (CGO) 本地 ASR · SQLite · YAML 配置。

## ② 创作思路

### 灵感来源

我自己就是一个会被「时间黑洞」吞掉的人：打开文档查个 API，半小时后才回过神；一天结束时被问「今天做了啥」，只能凭模糊记忆拼凑。市面上的时间追踪工具要么只数时长不归类，要么数据上云让我不放心。

与此同时，语音输入虽然快，但转出来全是「嗯、啊、就是说」，发到工作群里显得很不专业；想用 AI 帮写日报，可工作数据散落在十几个 App 里，AI 根本看不到。

### 解决的痛点

| 😖 痛点 | ✨ Shadow Worker 的做法 |
|---------|------------------------|
| 说不清时间花哪了，写日报靠回忆 | 后台自动记录应用时段并归类，时间轴一目了然 |
| 想输入一句话，切输入法打字打断思路 | 全局热键长按即录，松开出文字，一键注入焦点输入框 |
| 语音转文字有口语残留，发出去不专业 | ASR 结果自动经 LLM 润色，去口语、补标点 |
| 想让 AI 写日报，但数据散落各处 | 内置 MCP 服务，AI Agent 可直接查询工作日志 |

### 为什么选这个方向

- **场景真实**：这是每天都在发生的、高频的、有明确付费意愿的场景。
- **差异化**：把「时间追踪 + 语音输入 + AI 摘要 + MCP」四件事拧成一个本地优先的整体，市面上没有同类整合产品。
- **契合「学习工作」赛道**：直接服务知识工作者的核心工作流。

## ③ 体验方式

### 方式一：直接打开 HTML 原型（本 Demo）

本仓库的 `ui-wireframe-v2.html` 是一个**完整可交互的产品原型**（纯前端，无依赖），用 TRAE Work 生成：

1. 双击打开 `ui-wireframe-v2.html`（任意现代浏览器）。
2. **顶部「作品介绍区」**：项目定位、核心功能、痛点/价值、技术栈、4 张 TRAE 开发截图。
3. **左侧导航**切换四个主视图：
   - **概览**：工作时长统计、热力图、类别占比排行、应用排行、快捷开关。
   - **时间轴**：日视图时间条（编程/沟通/浏览/离开分段）、工作记录列表、事件列表、日历选择。
   - **设置**：6 个 Tab 全可交互——语音输入（热键/ASR 模型/麦克风测试）、行为采集（白名单/选窗口遮罩）、图像识别（VLM）、文字润色（LLM/提示词）、个人提示词（快捷键绑定）、快捷工具。
   - **系统**：关于、开机自启、MCP 服务（含 Claude/Cursor 接入配置）、软件更新。
4. **第 2 节「录音气泡」**：点击下方按钮依次体验 开始录音 → 实时识别 → 结束并润色 → 完成 四个状态；可切换「自动润色」开关观察 AI 润色图标变化。

> 📦 **打包提交**：将 `ui-wireframe-v2.html` 与 `TRAE-images/` 文件夹一起打包成 zip（<20M）上传即可。

### 方式二（可选）：运行真实应用

Shadow Worker 已有可运行的工程实现（Go 后端 + Qt 客户端），编译后可体验真实 ASR / 行为采集 / MCP 能力。原型用于产品设计对齐与功能演示。

## ④ TRAE 实践过程

本 Demo（HTML 原型）完全在 **TRAE Work** 中通过对话驱动完成设计与生成。

### 开发流程

1. **需求拆解**：在 TRAE Work 中用自然语言描述 Shadow Worker 的产品形态、目标用户和核心模块，让 AI 先产出数据字段定义与整体信息架构。
2. **逐页生成**：通过多轮对话驱动 TRAE 依次生成 概览页 → 时间轴页 → 设置页（6 个子 Tab）→ 系统页，右侧实时预览交互效果，所见即所得。
3. **复杂交互打磨**：针对「选窗口遮罩」「录音气泡四状态机」「MCP 配置复制」等复杂交互，用自然语言反复描述需求，让 TRAE 调整到符合预期。
4. **整体联调**：TRAE 一键生成完整可交互的 HTML 原型，作为后续 Go/Qt 工程实现的设计与开发对齐依据。

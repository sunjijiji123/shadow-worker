## 1. 配色系统(全局统一)

### 1.1 基础色板(深色主题)

```
背景层:
  --bg     #18181B   应用主背景(近黑)
  --bg2    #232327   侧边栏/卡片间背景
  --bg3    #2C2C31   卡片背景
  --rule   #3F3F46   边框/分隔线

文字层:
  --ink    #F4F4F5   主文字(亮)
  --muted  #9CA3AF   次要文字(标签/说明/时间戳)

强调色:
  --accent         #10B981  绿(accent-dim #059669)
                   ← v2 用绿,不是蓝!所有选中/激活/按钮主色用绿
  --danger         #EF4444  红(删除/错误/警告徽章)
```

**关键:v2 用绿色作 accent(#10B981),与 coding 类别色撞色,但角色不同。**
- accent 绿 = UI 操作色(选中态/主按钮/状态就绪)
- coding 蓝(#3B82F6) = 类别色(timeline/统计)
- 不要混用。

### 1.2 类别固定色(timeline/统计/卡片统一)

```css
.coding  { background: #3B82F6; }  /* 编程 */
.office  { background: #8B5CF6; }  /* 办公 */
.browser { background: #F59E0B; }  /* 浏览 */
.chat    { background: #10B981; }  /* 沟通 ← 注意:与 accent 同色值 */
.other   { background: #6B7280; }  /* 其他 */
.idle    { background: #9CA3AF; }  /* 离开/静止 */
```

### 1.3 事件类型色(timeline 上方标记 + 事件列表)

```
voice     #10B981 绿   ▲ 语音
prompt    #F59E0B 橙   ◆ 提示词注入
screenshot #3B82F6 蓝  ◉ 截图
```

### 3.2 元素

**stats-row(4 个等宽统计卡):**
- 今日工作时长(大数字 28px 700 字重)+ 较昨日 delta(绿正/红负)
- 打断次数 + 较昨日 delta(**打断 = active↔idle 切换次数**,见 grpc-mcp-api.md)
- 涉及应用数 + 类别明细(coding / browser / chat)
- 当前应用(文字)+ 类别 + 采集状态

**采集应用卡(横向滚动):**
- 每个白名单应用一个小卡(40px 图标 + 名 + 类别),末尾"+ 添加"
- [管理]按钮 → 跳转设置页"行为采集"tab

**活跃热力图卡:**
- GitHub 贡献格风格,横向滚动(可显示多个月)
- 每格 = 1 天,绿色深浅 5 档表示当日活跃度(0→满格)
- hover tooltip 显示"日期 + 活跃分钟"
- 右上角图例"较少 ▒▒▒▒▒ 较多"

**快捷开关卡:**
- 采集状态(开关,即 Pause/Resume)
- 开机自启(开关)
- 启动即采集(开关)
- 切换后自动保存并 toast"已保存"

**类别占比排行卡:**
- 每行:类别圆点 + 名称 + 横条(类别色)+ "占比% · 时长"
- 按占比降序
- 说明文字随范围切换("今日/本周/本月各类别活跃时长占比")

**应用排行卡:**
- 每行:应用名 + 横条(类别色,宽度=相对时长)+ 时长
- 按时长降序

### 3.3 顶部操作

- [今天][本周][本月] chip-group → 调 `GetOverview(range)`,刷新全部统计卡/热力图/排行
- [刷新] ghost 按钮 → 重新拉 `GetOverview`

### 3.4 数据来源

- `OverviewService.GetOverview(range)` — stats 卡 + 应用排行 + 状态
- `OverviewService.WatchOverview`(stream)— 实时推今日时长/采集状态/当前应用
- `OverviewService.GetHeatmap(months_back)` — 活跃热力图
- `OverviewService.GetCategoryRank(date/range)` — 类别占比排行
- 快捷开关:`CollectionService.Pause/Resume` + `ConfigService.SaveConfig`

> 旧版"运行状态卡"(4 个状态点)**已移除**,状态信息融入"当前应用"卡的副文字。proto gap 详见 grpc-mcp-api.md 与 IMPLEMENTATION-PLAN.md §1.4。

## 4. 时间轴页(Timeline)

> **v2 取消了周/月视图**。时间轴只保留**单日视图**,由日期选择器驱动。
> 周/月的统计能力只在概览页(今天/本周/本月切换)体现。

### 4.1 顶部

```
时间轴    [< 2026-06-18 周四 >] [今天]
           ↑─── 日期选择器 ───↑
           点击中间日期展开日历弹窗
```

**日期选择器(date-picker):**
- `<` `>` 前后翻一天
- 中间显示"YYYY-MM-DD 周X",点击展开**日历弹窗**(calendar-popup)
- 日历弹窗:月份切换(`<` `>`)+ 月份标题 + 星期表头(日一二三四五六)+ 日期网格
  - 今日:绿框(未选中时)
  - 选中日:绿底黑字
  - 有数据日:底部小绿点
  - 点击日期 = 切换到该日 + 关闭弹窗
- [今天] ghost 按钮 → 一键跳回今天

### 4.2 日视图

```
2026-06-18 周四    工作 3h 20m · 活跃 4 段

09:00 10:00 11:00 12:00 13:00 14:00 15:00 16:00 17:00 18:00
                                                        ↑ ruler

  ▲      ◆         ◉              ▲                      ↑ events 层
(语音)(提示词)(截图)              (语音)                  在 track 上方

[coding ████████][idle ██][coding ███████][chat ██][browser ████][coding ██]
 ↑ 横向单焦点轴,按时间段拼色块,类别固定色,idle 灰间隙

图例:■编程 ■沟通 ■浏览 ■办公 ■其他 ■离开    ▲语音 ◆提示词 ◉截图
```

**规则(锁定):**
- 单焦点轴(不开多行)
- 色块按类别上色(第 1.2 节)
- idle 是灰色间隙段(.idle class)
- 事件标记在 track **上方**,鼠标 hover 显示 title(时间+内容)
- ruler 时间刻度横排

### 4.3 列表卡(工作记录 / 事件列表 双 tab,视图下方常驻)

列表卡顶部有双 tab 切换:**工作记录** | **事件列表**,两个 tab 都带类别/类型筛选 chip。

## 5. 设置页(Settings)

### 5.1 顶部 tab-strip(6 个 tab)

```
语音输入 | 行为采集 | 图像识别 | 文字润色 | 个人提示词 | 快捷工具
   ●                                                    ← active 下划线 --accent
```

> 顺序与 v2 HTML `settings-tabs` 一致。"行为采集"是独立 tab(白名单 + 采集规则),不再混在图像识别里。

### 5.2 语音输入 tab

**3 张卡:**
1. **录音热键** — 总开关 toggle + radio(长按录音 / 按一次录音)+ 修饰键 select + 按键 input
2. **ASR 模型服务** — 模型 chip(小米-ASR/DeepSeek-ASR/本地-whisper/+添加)
   - 云端表单:服务商名/Base URL/模型/识别语言/API Format/认证方式/API Key + [测试连接]
   - 本地表单:模型路径(浏览…)+ 模型名(从路径识别)+ 识别语言
   - 切 chip 查看编辑对应配置;[+ 添加模型]弹窗(显示名/服务商/部署类型/自定义名)
3. **音频设备** — 输入设备下拉 + [测试麦克风](测试中变红+脉冲点)+ 音量条

### 5.3 行为采集 tab

**2 张卡:**
1. **采集应用(白名单)** — [+ 添加](选窗口遮罩)+ [扫描应用]
   - 每个应用一行:40px 图标 + 名 + 类别 chip-group(coding/browser/chat/office/other,点击改类别)+ [× 移除]
   - 只有这里的应用才会被记录到时间轴;VLM 仅活动窗口模式下也只采集这些应用
   - 增删改**即时生效**(不等表单保存)
2. **采集规则** — 锁屏时暂停采集(toggle)+ 空闲超时阈值(分钟 input)

### 5.4 图像识别 tab

**4 张卡:**
1. **VLM 屏幕理解** — 总开关 toggle + radio(定时截图 / 按需截图)
   - 定时:定时间隔(分)input
   - 按需:修饰键 select + 按键 input
2. **VLM 模型服务** — 模型 chip(小米-VLM/DeepSeek-VL/本地-ollama/+添加),表单同 ASR(云端/本地两套字段)+ [测试连接]
3. **画面采集范围** — radio(整个屏幕 / 仅当前活动窗口)。仅活动窗口模式下,VLM 只采集白名单应用窗口
4. **采集参数** — 采样间隔(ms)/ 空闲超时(s)/ 精度(low/medium/high)

### 5.5 文字润色 tab

**3 张卡:**
1. **自动润色** — toggle 开关 + 说明(开启后 ASR 结果先经 LLM 润色)
2. **LLM 模型服务** — 模型 chip(DeepSeek-LLM/小米-LLM/本地-ollama/+添加),表单同 VLM + [测试连接]
3. **润色提示词** — textarea(系统默认 + 用户自定义,存 config.yaml)

### 5.6 个人提示词 tab

**2 张卡:**
1. **快捷提示词注入** — toggle 开关 + 前缀键 select(Ctrl/Alt/Win)
2. **个人提示词列表** — [+ 添加]
   - 每条:名称 input + 快捷键(前缀键 cap + 数字/字母 key-input)+ 删除 + textarea 内容 + hint"按 Ctrl+1 注入"

### 5.7 快捷工具 tab

**2 张卡:**
1. **桌面截图** — 修饰键 + 按键 + 保存位置 + [立即截图]
2. **数据管理** — [打开数据目录] [清空数据]

### 5.8 底部统一保存条(所有 tab 共享)

```
                                                       [保存]
```

- 只有 **[保存]**,无"重置默认"
- 浮在内容区底部(left:180px 从侧边栏右侧开始),渐变背景遮住下方内容
- 点击 → 保存全部表单配置到 Go config.yaml + toast"设置已保存"
- **白名单不在保存条范围**(即时生效);概览页的快捷开关切换自动保存 + toast"已保存"

## 6. 系统页(System)

**6 张卡:**
1. **关于** — 版本号 + 仓库地址
2. **开机自启** — toggle 开关(登录 Windows 后自动启动到托盘)
3. **MCP 服务** — 状态灯(运行中)+ [重启 MCP]
   - 可用工具列表(4 个):`get_worklog` / `get_summary` / `search_events` / `list_apps`(每个带说明)
   - 接入配置:三个 tab(Claude Desktop / Cursor / 原始 JSON)+ 代码框 + [复制]按钮
4. **软件更新** — 当前版本 + 上次检查时间 + 状态徽章(更新服务可用/不可用)+ [官方网站/GitHub/更新日志/检查更新] + 启动时检查 toggle + 每日检查 toggle + 更新服务器地址 input
5. **数据管理** — 本地 SQLite + 截图缓存,[打开数据目录] [清空所有记录]
6. **配置文件路径** — 只读 input 显示 `%APPDATA%\shadow-worker\config.yaml`


**AI 润色图标(右上角):**

- hover tooltip:"AI 润色"

**操作:**
- Enter = 注入(到焦点输入框)
- Esc = 关闭气泡
- [复制] [关闭] 按钮

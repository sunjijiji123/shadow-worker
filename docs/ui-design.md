# Shadow Worker UI 设计规范

> 本文档收录架构讨论中**已经定死**的 UI 设计决策。
> Trae/AI 照此实现,不添加未经讨论的新交互。
>
> 覆盖:概览页 / Timeline / 白名单管理 / 浮窗气泡 / 设置页结构。
> 类别固定色映射、数据来源、hover 逻辑等细节都在这里。

## 0. 全局规范:类别固定色映射

**所有 UI(卡片 / Timeline / 统计 / 状态)统一用这套色,不可改。**

| 类别 | 色值 | 含义 |
|------|------|------|
| `coding` | `#3B82F6` 蓝 | 代码编辑器(Cursor/VSCode/JetBrains) |
| `office` | `#8B5CF6` 紫 | 办公(Excel/Word/PPT) |
| `browser` | `#F59E0B` 橙 | 浏览器(Chrome/Edge) |
| `chat` | `#10B981` 绿 | 沟通(微信/钉钉) |
| `other` | `#6B7280` 灰深 | 其他白名单应用 |
| `idle` | `#9CA3AF` 灰浅 | 离开/静止(不归任何应用) |

## 2. Timeline(横向色块时间轴)

**整个应用最复杂的一块。** 十几轮架构讨论才定死,严格按此实现。

### 2.1 形态(锁定)

```
日期  [今天 v]   [日] [周] [月]    ← 顶部:日期切换 + 视图模式

09:00 ─────── 10:00 ─────── 11:00 ─────── 12:00 ─────── 13:00 ───
                                                
事件层  ▲讨论架构          ▲确认需求              ▲截图        ← 点状事件,轴上方
        │语音              │提示词                │截图          hover 看内容
        
色块条  ████████████████░░████████░░████████████████████████████░░██
        └─Cursor(蓝)──┘│  │       │  └─微信(绿)─┘└─浏览器(橙)─┘│
                      灰  灰         ↑ 单焦点轴:焦点切换直接换色块
                      idle idle        不开多行

图例  ■编程(蓝) ■沟通(绿) ■浏览(橙) ■其他(灰深) ░离开(灰浅)
      ▲语音 ◆提示词 ◉截图
```

### 2.2 规则(逐条)

| 维度 | 决策 |
|------|------|
| 形态 | **横向单焦点轴**,分钟级色块(**不开多行**) |
| 上色 | **按应用类别固定色**(见第 0 节),不按应用名 |
| idle | **灰色间隙**,单焦点轴上直接留空(不开新行) |
| 事件 | **轴上方标记**:`▲`语音 / `◆`提示词 / `◉`截图,hover 看内容 |
| 范围 | 默认今天,**可切周/月视图** |
| 多应用 | 焦点切了色块就换色,**不开多行** |

### 2.3 数据来源(关键)

| 元素 | DB 表 | 写入时机 |
|------|-------|---------|
| 色块段 | `activity_segments` | **状态变化时**(app 切换 / active↔idle),不是每帧 |
| 事件标记 | `events` (`type=voice/prompt_inject/screenshot`) | 发生时立即写 |

**关键设计:DB 写入频率极低。** 不是每 300ms 写一条——只在状态变化时开新段、同段内只更新 `end_ts`。一天几十到几百条 segment,SQLite 毫无压力。

### 2.4 hover tooltip 内容规则(分两类)

```
coding 段:
  summary = null
  hint    = "use code-worklog skill"   ← 主动提示 agent 去补
                                           (内容不在我们这边,在 agent 侧)
  tooltip 显示: "编程 · {project}(如有) · 点击让 AI 补全详情"

其他段(browser/office/chat/other):
  summary = VLM 摘要(若已生成)或 null(等待生成)
  tooltip 显示: 摘要文本,无摘要显示"未生成日报,点击生成"
```

### 2.5 周月视图(聚合,不存新数据)

```
周视图:  每天一列
         显示 "编程 Xh / 沟通 Yh / 浏览 Zh"
         SQL: GROUP BY category, 按天
         用途: 看一周分布趋势

月视图:  热力图
         每天一个小格子,颜色深浅 = 当日总工作时长
         SQL: SUM(today_minutes) GROUP BY date
         用途: 看月度活跃趋势
```

**数据复用:DB 只存日级原始数据,周月靠 SQL 聚合。**

### 2.6 "非白名单应用完全不存在"

Timeline 上**只显示白名单应用的段**。非白名单应用被 movement 跳过,DB 不留任何痕迹。这是隐私的根本保障,不是过滤,是从采集源头就不记录。

### 2.7 gRPC 接口

- 走 `CollectionService`(见 `proto/collection.proto`)拿状态
- Timeline 数据走新增的 timeline 查询接口(如 `proto` 已有;若没有,Trae 在 collection.proto 加 `GetTimeline(date)` RPC)

## 3. 白名单管理(设置页 → 采集应用)

### 3.1 卡片式列表

```
采集应用                                  [+ 添加应用]
本机行为采集只追踪以下应用,其他完全不记录。

┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│ ◆ Cursor    │  │ 🌐 Chrome   │  │ 📊 Excel    │
│             │  │             │  │             │
│ 编程        │  │ 浏览        │  │ 办公        │  ← 类别 chip(点改)
│             │  │             │  │             │
│ 今日 2h15m  │  │ 今日 1h     │  │ 今日 45m    │  ← 今日时长
│ [改类别][删] │  │ [改类别][删] │  │ [改类别][删] │
└─────────────┘  └─────────────┘  └─────────────┘

┌─────────────┐
│   + 添加    │
│  点选桌面   │
│   窗口      │
└─────────────┘
```

### 3.2 卡片元素

- **应用图标**:从 exe 提取
- **应用名**
- **类别 chip**:固定色,点击弹 5 选项改类别
- **今日时长**:实时刷新(让用户感知"在采")
- **[改类别] / [删除]**

### 3.3 添加流程(选窗口交互,复用截图技术栈)

```
点[+ 添加应用]
    ↓
全屏半透明遮罩(80% 黑,复用 screenshotwindow 遮罩机制)
    │
    │ 鼠标移动 → WindowFromPoint 拿窗口句柄
    │ → 该窗口蓝色描边高亮 + 显示标题 "Cursor - main.go"
    │
    │ 点击选中 → 遮罩消失
    ↓
抓进程信息:
  GetWindowThreadProcessId(pid)
  QueryFullProcessImageName → "C:\...\Cursor.exe"
  ExtractIcon → 图标
    ↓
弹确认卡:
┌──────────────────────────┐
│ ◆ Cursor                  │
│ C:\Users\...\Cursor.exe   │
│ 类别: [其他 v]            │  ← 默认 other,可改
│ [取消]      [添加]        │
└──────────────────────────┘
```

### 3.4 规则

| 决策 | 说明 |
|------|------|
| 标识 | **完整进程路径**(主键),路径变(搬家/升级)会失效,需重选 |
| 类别时机 | **添加时可选**,默认 other;之后随时点 chip 改 |
| 生效方式 | **白名单增删改即时生效**(不等表单保存);表单配置才手动保存 |
| 初始化 | **空白开始**,无预置规则;遇到新应用主动问 |

### 3.5 改类别交互

点卡片上的类别 chip → 弹 5 个选项(带固定色)→ 选一个 → 立即生效。

### 3.6 gRPC 接口

走 `WhitelistService`(见 `proto/whitelist.proto`):
- `List / Add(path,name,category) / Remove(path) / UpdateCategory(path,category)`

### 3.7 Win32 调用

选窗口交互的 Win32 调用见 `docs/win32-api.md` 第 3 节。
关键:`WindowFromPoint` / `GetWindowRect`(画高亮)/ `QueryFullProcessImageNameW`(抓路径)。

## 4. 浮窗 + 录音气泡

### 4.1 浮窗(仅录音时出现)

```
【录音中】(只有浮窗,右下角)
                                          ┌──────────────┐
                                          │ ▮▮▮▮▮ 录音中… │  ← 纯频谱
                                          └──────────────┘
   frameless 置顶,不抢焦点
```

**职责边界:仅录音时出现**。不是常驻控制中心。
录音结束 → 浮窗短暂显示结果 → 3 秒后自动消失(仅 Auto 模式;Preview 模式气泡常驻)。

### 4.2 气泡(录音后,在浮窗上方)

```
【Preview 模式 · 气泡常驻】
                            ┌──────────────────────────┐
                            │ 帮我把这段代码重构一下…    │  ← 可编辑最终文本
                            │ [取消]      [复制]         │  ← 只这两个按钮
                            └──────────────────────────┘
                            ┌──────────────────────────┐
                            │ ✓ 识别完成                │  ← 浮窗(下方)
                            └──────────────────────────┘
   用户自己 Ctrl+V 到目标输入框
   点[取消] 或 ESC 关掉(= 关浮窗)

【Auto 模式 · 气泡临时】
   录音完瞬间,焦点还在原输入框(未动) → 直接注入
                            ┌──────────────────────────┐
                            │ 帮我把这段代码重构一下…    │
                            │ ✓ 已注入到焦点输入框        │
                            └──────────────────────────┘
   3 秒后自动消失
```

### 4.3 关键规则(几轮纠正后定死)

| 决策 | 说明 |
|------|------|
| 气泡位置 | **浮窗上方**(右下角垂直堆叠) |
| Preview 按钮 | **只有 [取消] [复制]**,**没有注入按钮**(焦点不可信) |
| Auto 注入 | 录音完瞬间直接注入(唯一焦点可信窗口),气泡闪现 3s |
| 润色 vs 注入 | **两个正交配置**:`polish.enabled`(on/off)+ `inject.mode`(preview/auto) |
| 气泡文本 | **永远是最终文本**(润色后或原文,取决于 polish 配置) |
| 文本可编辑 | Preview 模式气泡内 TextEdit,ASR 识别错可手改 |

### 4.4 焦点可信原理(为什么 Preview 不能注入)

> 气泡常驻期间用户会乱点,焦点不可信。点"注入"时焦点可能在任何地方。
> Auto 模式利用"录音瞬间焦点未动"的时间窗,这是唯一可信的注入时机。

详见 `docs/ARCHITECTURE.md` 第 9 节。

## 5. 设置页结构

### 5.1 三组分组 + 概览

```
┌─────────────────┬──────────────────────────────────┐
│ ⚙️ 设置          │                                  │
│                 │                                  │
│ ▼ 采集          │   ← 分组1: 决定"采什么"           │
│   📋 采集应用    │                                  │
│   📊 采集参数    │                                  │
│   📷 屏幕理解    │                                  │
│                 │                                  │
│ ▼ 输入          │   ← 分组2: 决定"语音怎么用"       │
│   🎤 语音识别    │                                  │
│   🎙 音频设备    │   ← 新增:麦克风选择(Qt 本地存)   │
│   ✨ 文本润色    │                                  │
│   ⌨️ 注入模式    │                                  │
│   ⌨️ 快捷键      │                                  │
│                 │                                  │
│ ▼ 系统          │   ← 分组3: 外部接口/数据/通用      │
│   🔌 MCP 服务    │                                  │
│   📁 数据管理    │                                  │
│   ℹ️ 关于        │                                  │
└─────────────────┴──────────────────────────────────┘
```

### 5.2 配置项清单(按组)

**采集组:**
- 📋 采集应用 → 白名单卡片管理(见第 3 节)
- 📊 采集参数 → 精度下拉(low/medium/high,展开=4 旋钮)、采样间隔、静止超时
- 📷 屏幕理解 → 触发模式(scheduled/on_demand/off)、provider、定时频率

**输入组:**
- 🎤 语音识别 → cloud/local radio + provider CRUD + 测试连接
- 🎙 音频设备 → 设备下拉(系统枚举)+ 测试电平(录 3 秒看波形)
- ✨ 文本润色 → enabled 开关 + provider + 全局 prompt
- ⌨️ 注入模式 → preview/auto radio
- ⌨️ 快捷键 → 录音/截图/提示词热键

**系统组:**
- 🔌 MCP 服务 → 开关 + 配置摘要(显示 agent 该写的命令)
- 📁 数据管理 → 数据路径 / 导出 / 清空
- ℹ️ 关于 → 版本 / 更新

### 5.3 配置存储分层(锁定)

```
Go 配置(走 gRPC,存 YAML config.yaml):
  asr / polish / inject / movement / vlm / mcp / hotkeys / general

SQLite(走 gRPC,存结构化数据):
  app_categories(白名单)/ activity_segments / events

Qt 本地(QSettings,不走 gRPC):
  audio/device_id        ← 麦克风设备 ID
  ui/float_window_pos    ← 浮窗位置
  ui/main_window_geometry← 主窗口尺寸/位置
  ui/last_settings_tab   ← 上次打开的设置页
```

**原则:Go 永远不碰麦克风设备 ID,Qt 永远不碰业务配置。**

### 5.4 生效方式

| 配置类 | 生效方式 |
|--------|---------|
| 白名单增删改 | **即时生效**(不等保存) |
| 其他表单配置 | **手动保存**(底部 [保存] [取消]) |

### 5.5 精度档位映射(给"采集参数"页用)

```
低:  thresh=50, ratio=0.005, 采样 500ms, idle 超时 15s
中:  thresh=30, ratio=0.002, 采样 300ms, idle 超时 10s   ← 默认
高:  thresh=15, ratio=0.001, 采样 200ms, idle 超时 5s
```

普通用户给下拉三档,高级用户展开调原始 4 旋钮。

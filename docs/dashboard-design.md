## 4. 区块详解

### 4.1 今日工作卡片(左上,主信息)

```
┌─────────────┐
│ 今日工作      │  ← text-secondary 标签
│  2h 15m      │  ← 大号 text-primary 数字(32-40px)
│  活跃 4 段    │  ← text-secondary 小字
│  3 个应用     │
└─────────────┘
```

**数据来源:**
- 工作时长 / 活跃段数 / 应用数 → `activity_segments` 当天聚合
- gRPC:`OverviewService.GetOverview` → `today_minutes / active_segments / apps.length`

### 4.2 状态卡片(右上)

```
┌─────────────────────────┐
│ 状态                      │
│ ● 采集运行中  [⏸ 暂停]    │  ← 绿点 success + 暂停按钮
│ 🎤 语音 就绪              │  ← 绿点
│ 📷 VLM  就绪              │
│ 🔌 MCP  运行中             │
│ 💾 DB   就绪              │
└─────────────────────────┘
```

**状态灯语义:**

| 状态 | 颜色 | 含义 |
|------|------|------|
| running / ready / 就绪 | 🟢 success `#10B981` | 正常工作 |
| paused / off / 未启动 | 🟡 warning `#F59E0B` | 暂停或未开 |
| error / 错误 | 🔴 error `#EF4444` | 出错(详情 hover) |
| unconfigured | ⚪ text-muted | 未配置 |

**交互:**
- 暂停按钮:调 `CollectionService.Pause`,变恢复按钮
- 状态灯 hover 显示详情(如错误信息)

**数据来源:** `OverviewService` → `collection_status / asr_status / vlm_status / mcp_status`

### 4.3 应用时长条(中)

```
┌────────────────────────────────────────────┐
│ 应用时长                                     │
│ Cursor ████████████ 1h30m    ← 蓝条 coding  │
│ Chrome █████ 30m             ← 橙条 browser │
│ VSCode ███ 15m               ← 蓝条 coding  │
└────────────────────────────────────────────┘
```

**规则:**
- 按当日时长降序
- 条的颜色 = 应用类别固定色(第 2.2 节)
- 条长 = 该应用时长 / 当日最长应用时长(归一化)
- 只显示白名单应用(非白名单不存在)

**数据来源:** `activity_segments` GROUP BY app_name,当天

### 4.4 类别分布条(下)

```
┌────────────────────────────────────────────┐
│ 类别分布                                     │
│ ████████████░░░░░░░  编程 65%   ← 蓝        │
│ ████░░░░░░░░░░░░░░  浏览 20%   ← 橙        │
│ ██░░░░░░░░░░░░░░░░  其他 15%   ← 灰        │
└────────────────────────────────────────────┘
```

**规则:**
- 按类别聚合(非按应用)
- 条长 = 该类别时长 / 当日总时长(百分比)
- 类别色固定(第 2.2 节)
- idle 不计入分布(只看有效工作)

**数据来源:** `activity_segments` GROUP BY category,当天

## 6. 刷新机制

| 区块 | 刷新方式 |
|------|---------|
| 今日工作卡 | gRPC `OverviewService.WatchOverview`(ServerStream)实时推 |
| 状态卡 | 同上 |
| 应用时长条 | 同上(或定时 30s 轮询) |
| 类别分布 | 同上 |

**实现:Qt ViewModel 订阅 `WatchOverview` stream,收到 `OverviewUpdate` 就 emit 信号刷新 QML。**

## 7. 空状态(无数据时)

首次启动 / 当天还没采集:
```
┌─────────────┐
│ 今日工作      │
│  0m          │  ← 大数字 0
│  暂无记录     │  ← 提示
│  开始添加应用  │  ← 链接到 Whitelist 页
└─────────────┘
应用时长条:显示"暂无数据,去白名单添加应用"
类别分布:不显示
```

## 8. 数据流(完整)

```
Go 后台 collector 每 300ms 采样
  → 状态变化时写 activity_segments
  → 定期触发 OverviewService 推送(或 Qt 主动 GetOverview)
      ↓ gRPC
Qt DashboardViewModel
  → Q_PROPERTY today_minutes / active_segments / apps / statuses
  → emit dataChanged
      ↓ 信号槽
QML DashboardPage
  → 大数字 / 状态灯 / 应用时长条 / 类别分布 自动刷新
```
